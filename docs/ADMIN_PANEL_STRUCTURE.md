# ADMIN_PANEL_STRUCTURE.md — Logist.kg

## 1. Назначение админ-панели

Админ-панель нужна для управления платформой Logist.kg, пользователями, грузами, жалобами, рекламой и безопасностью.

---

## 2. Основные разделы

### Dashboard
Главная страница админки.

Показывает:
- количество пользователей
- количество водителей
- количество отправителей
- количество активных грузов
- количество откликов
- количество жалоб
- количество неподтверждённых документов
- количество платных объявлений
- базовую статистику

---

### Users
Управление всеми пользователями.

Подразделы:
- все пользователи
- водители
- отправители
- диспетчеры
- администраторы
- заблокированные
- неподтверждённые

Действия:
- открыть профиль
- проверить документы
- подтвердить пользователя
- заблокировать
- разблокировать
- изменить роль
- посмотреть историю действий

---

### Companies
Управление компаниями и отправителями.

Действия:
- просмотр профиля компании
- проверка реквизитов
- подтверждение компании
- блокировка
- просмотр грузов компании

---

### Drivers
Управление водителями.

Действия:
- просмотр профиля
- проверка паспорта
- проверка техпаспорта
- просмотр транспорта
- подтверждение
- блокировка
- рейтинг и отзывы

---

### Freights
Управление грузами.

Действия:
- просмотр всех грузов
- фильтр по статусу
- фильтр по маршруту
- фильтр по дате
- фильтр по компании
- открыть карточку груза
- скрыть груз
- удалить груз
- отметить как спам
- отметить как VIP / платное объявление

Статусы:
- active
- in_review
- blocked
- completed
- cancelled

---

### Bids
Управление откликами.

Действия:
- просмотр откликов по грузу
- просмотр откликов по водителю
- скрытие подозрительных откликов
- удаление спам-откликов

---

### Reports
Управление жалобами.

Действия:
- просмотр жалобы
- открыть объект жалобы
- отметить как обработано
- заблокировать пользователя
- скрыть груз
- удалить спам
- оставить комментарий модератора

Статусы:
- new
- in_progress
- resolved
- rejected

---

### Verification
Проверка документов.

Подразделы:
- документы водителей
- документы компаний
- телефоны на подтверждении
- неподтверждённые аккаунты

Действия:
- одобрить
- отклонить
- запросить повторную загрузку
- оставить комментарий

---

### Chat Moderation
Контроль чатов.

Нужно для:
- жалоб на сообщения
- жалоб на спам
- блокировки нарушителей

Действия:
- открыть диалог
- скрыть сообщение
- заблокировать отправителя

---

### Payments
Управление оплатами.

Нужно для:
- платных объявлений
- подписок
- рекламы

Действия:
- просмотр платежей
- фильтр по статусу
- фильтр по пользователю
- просмотр тарифа
- ручное подтверждение платежа
- возврат / отмена при необходимости

---

### Subscriptions
Управление подписками.

Действия:
- активировать подписку
- отключить подписку
- продлить
- проверить срок действия

---

### Ads
Управление рекламой.

Действия:
- создать рекламу
- редактировать рекламу
- включить / выключить
- назначить позицию
- назначить срок показа

---

### Banners
Управление баннерами.

Действия:
- создать баннер
- редактировать баннер
- включить / выключить
- выбрать страницу показа

---

### Notifications
Системные уведомления.

Действия:
- отправить массовое уведомление
- отправить уведомление пользователю
- уведомить только водителей
- уведомить только компании

---

### Audit Logs
Логи действий.

Показывает:
- кто что изменил
- кто кого заблокировал
- кто подтвердил документы
- кто удалил груз
- кто изменил рекламу

---

### System Settings
Настройки платформы.

Примеры:
- лимит объявлений в день
- лимит откликов в час
- включение/выключение аукциона
- минимальный рейтинг
- валюты
- страны
- настройки модерации
- контакты платформы

---
### Настройки социальных сетей

Администратор может добавлять и редактировать ссылки:

- Telegram
- WhatsApp
- Instagram
- Facebook
- YouTube
- TikTok
- Email поддержки
- Телефон поддержки

Эти ссылки используются:
- в футере сайта
- на странице контактов
- в мобильной версии

---

### Настройки внешнего вида сайта (Theme Settings)

Администратор может менять внешний вид платформы без изменения кода.

Доступные настройки:

**Тема сайта**
- светлая
- тёмная
- авто

**Фон сайта**
- цвет фона
- изображение фона
- прозрачность

**Цвета интерфейса**
- основной цвет
- цвет кнопок
- цвет ссылок
- цвет hover

**Шрифты**
- основной шрифт сайта
- размер шрифта
- межстрочный интервал

**Сайдбар**
- цвет сайдбара
- ширина сайдбара
- стиль иконок
- включить/выключить иконки

---

### UI настройки

Можно управлять:

- логотипом сайта
- favicon
- изображением на главной странице
- баннерами
- текстами на главной странице
- кнопками CTA

---

### Хранение настроек

Все настройки хранятся в коллекции:
system_settings/main

Пример документа:

```json
{
  "siteTheme": "dark",
  "primaryColor": "#0A84FF",
  "fontFamily": "Inter",
  "backgroundImage": "url",
  "telegram": "https://t.me/logist",
  "instagram": "https://instagram.com/logist",
  "supportEmail": "admin@logist.kg",
  "supportPhone": "+996509139129"
}

- verification
- chat moderation

### finance_manager
Ограниченный доступ:
- payments
- subscriptions
- ads

## 6. Важные правила для админки

- все действия админа писать в audit_logs
- удаление грузов лучше делать мягким через status
- документы не удалять сразу, а архивировать
- доступ в админку только по role
- критические действия подтверждать повторно
- массовые действия ограничивать
